# 联动EMBY库筛选下载

以 Emby 的实际 `MediaSources` 建立本地版本库存，再通过 MoviePilot 已配置站点搜索并提交下载。插件不会直接读取或修改 Emby 数据库文件；仅在新的更高码率 WEB-DL 已入库、文件存在且被 Emby 识别后，才会精确清理同分辨率的旧 WEB-DL。

## 主要能力

- 全页采用统一的紧凑型媒体控制台视觉，页头、导航、卡片、表格、筛选栏与设置表单共用同一套间距、圆角和主题色，并适配桌面、手机、亮色与暗色主题。
- 同步一个或多个 Emby 服务和指定媒体库，电影按影片、电视剧按单集记录每个媒体版本。
- 记录 Emby 返回的真实入库时间，版本库存默认将最近入库的媒体排在最上方。
- 识别原盘、DIY、Remux、Encode、WEB-DL、DV、HDR10+、HDR10、HLG、分辨率、编码和码率。
- 质量类型、动态范围和分辨率按配置中的选择顺序决定自动择优优先级，码率可选高到低、低到高或不参与排序。
- 可分别设置 4K 与 1080P 最低种子体积（GB，0 表示不限制）；保存后会立即重算已扫描种子池，后续扫描、目标匹配、自动下载和最终提交也会再次校验。
- 维护目标清单，可为目标覆盖站点、季、版本数、保存路径和自动下载开关。
- 一个目标清单对应一个完整的 MoviePilot 推荐榜单或扩展推荐源，例如“豆瓣电影 Top 250”；创建时自动读取全部分页并导入榜单中的所有电影，而不是只添加单部影片。
- 目标清单内逐部以海报展示，Emby 已入库项目显示绿色勾选，并统计已入库与待补数量。
- 新目标清单保存后会立即用榜单内全部影片匹配最近一次已扫描种子池，按榜单顺序、站点、年份、质量顺序和码率规则生成目标候选；自动下载开关开启时按每轮数量限制优先提交这些匹配项。
- 默认开启纯数据汇总通知，覆盖 Emby 库存、目标清单、种子池扫描、下载任务和异常任务；通知统一使用 Emoji 排版，不展示影片、种子、路径或错误详情，并支持一键测试 MoviePilot 通知链路。
- 默认开启插件级代理，站点搜索、全站分页、详情解析和种子文件下载统一使用 MoviePilot 系统代理；关闭后这些站点请求强制直连，Emby 局域网连接不受影响。
- 每个目标可选择优先使用已扫描种子池；符合目标标题、年份、站点和质量筛选的候选会在自动下载时置顶，并沿用目标版本数和保存路径。
- 目标清单会标记已入库、等待入库或库存未同步，并显示库存记录数；每个目标可快速搜索并直接跳转到站点候选页，Emby 定时或手动同步后会刷新海报勾选状态。
- 按 UBits 的 WEB-DL、Remux、DIY 原盘、Encode 四个电影分类逐页扫描到末页，插件内固定每页 50 条；可按影片年份、媒体评分、种子体积、做种人数或码率升降序排列，保存后的同一规则也用于种子池自动下载顺序，未知值固定排在末尾。
- 全站种子池按四个质量分类独立分页展示，并显示各分类的合格候选数量，不再混合列表。
- 手动多选或“当前页全选”下载；自动模式支持每轮数量上限。
- 下载任务支持逐项复选、当前页全选、批量删除终态记录、重试已选失败或取消任务，以及一键重试全部失败任务；重试仍执行库存、重复种子、质量槽位和版本上限校验。同一种子已有有效任务、媒体已达到版本上限或已有相同质量槽位在下载时，插件会自动清理对应的旧失败记录，所有任务状态均以中文展示。
- 下载前以 SQLite `BEGIN IMMEDIATE` 原子占位，按“Emby 已有版本 + 下载中占位”执行最多 1–3 个版本的硬限制。
- 默认拒绝同一媒体的相同“质量类型 + 动态范围 + 分辨率”槽位，减少重复版本。
- WEB-DL 额外维护两个固定电影槽位：4K 最高码率版和 1080P 最高码率版。只要库存中已有任意 Remux 或 Encode，即使普通版本已达到 3 个，也会继续补齐缺少的 WEB-DL 槽位。
- 同分辨率 WEB-DL 已存在时，只允许码率更高且可识别的新版本建立升级任务；WEB-DL 槽位始终按码率从高到低选择，不受普通码率排序方向影响。站点标题没有码率时，扫描列表先以同片体积从大到小择优，媒体识别后再以文件体积和影片时长估算总码率用于升级判断。
- WEB-DL 升级必须先由 Emby 库存确认新版本、核对成功转移记录并确认文件存在；随后按精确路径删除旧媒体库软/硬链接、旧下载源文件和对应转移记录。任何校验或清理失败都会保留尚未安全清理的旧版本，并将任务标记为“旧版本清理失败”等待后续库存同步重试。
- 电影、电视剧和单个目标均可设置 MoviePilot 下载保存路径。
- 默认排除所有剧集；候选筛选和下载提交前都会再次校验。
- 四个分类均不设置页数上限，遇到末页或重复页才停止，并显示当前分类和累计页数。
- 可在定时任务设置中手动执行一次完整种子池流程；Cron 表达式会显示未来三次具体执行时间。
- 支持独立自动下载 Cron：不重新扫描，直接从最近一次种子池结果中按设定成功数量补位下载；全站扫描完成后仍会额外执行一轮。
- 自动下载会预先排除已处于 reserved、queued、downloading 或 present 状态的同一种子，提交时再做原子去重。
- 原盘、DIY、Remux、Encode、WEB-DL 和未知质量可分别设置保存路径；目标专用路径优先。

## 使用顺序

1. 在 MoviePilot 中配置并启用 Emby、下载器和站点。
2. 安装插件，在侧栏打开“EMBY库筛选下载”。
3. 在“规则设置”中选择 Emby 服务、媒体库、站点和质量条件，保存设置。
4. 先执行“同步 EMBY”，核对版本库存。
5. 使用“目标清单”精确补库，或刷新“站点种子池”浏览全站候选。
6. 自动下载保持关闭，完成规则验证后再按需开启。

## 重要边界

- 全选只作用于当前 50 条页面，不会隐式选择全部搜索结果。
- 无法识别媒体身份的种子不会提交下载，因为它无法可靠执行版本上限。
- 首次成功同步 Emby 版本库存之前，手动和自动下载都会被拒绝，防止在库存未知时盲目下载。
- 已提交到下载器的任务需在下载器中取消；插件只允许取消尚未提交的占位任务。
- 码率无法从标题识别时默认为未知。启用“设置码率时拒绝未知码率”可执行严格过滤。
- 插件数据保存在插件数据目录的 `library_download.db`，不写入 Emby 数据库。
